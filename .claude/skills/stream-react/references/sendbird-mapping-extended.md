# Sendbird -> Stream Chat React: extended long-tail mapping (Track S appendix 2)

The machine-generated long tail behind [`sendbird-mapping.md`](sendbird-mapping.md): ~520 symbol
rows for the Sendbird APIs that **no sampled real app used** but that an arbitrary customer
codebase might. Load order when migrating a symbol:

1. [`sendbird-mapping.md`](sendbird-mapping.md) - curated, verified/used rows. Check it first.
2. **This file - grep for the exact symbol; never read it end-to-end.** It is a lookup table,
   not reading material.
3. On a miss here too: the live docs per [`docs-map.md`](docs-map.md) and the installed types
   ([`../RULES.md`](../RULES.md) > Docs-first).

**Trust model.** Every row below is **INFERRED**: derived from the installed type definitions of
`@sendbird/chat` 4.21.1 / `@sendbird/uikit-react` 3.18.2 vs `stream-chat` 9.x /
`stream-chat-react` v14, then adversarially verified (independent refutation passes; disputed
rows re-checked against primary sources and corrected) - but **never exercised by a real app**.
Treat a row as a strong hypothesis: `npx tsc --noEmit` against the installed package outranks it,
and where a row names current stream-chat-react props/hooks, confirm on the matching
[`docs-map.md`](docs-map.md) page before building. Rows marked `- (gap)` with mapping `none` are
feature gaps - route them through the parity ledger and
[`sendbird-mapping.md`](sendbird-mapping.md) section 15's decision discipline (substitute /
rebuild / drop), never a silent TODO.

Family rows like `AIAgent*` (N symbols) collapse a whole symbol family that shares one verdict.
